Organizing Data with Tables

Table used to be the primary way that websites were built because CSS was not widely supported in browsers. All the structure we have accomplished so far was done primarily with tables. Now, table are specifically for organizing data as they should be.

Creating a Table

Tables are made up of data in columns and rows. At a minimum, the <table>, <tr>, and <td> elements must be used. Deeper semantic elements are available as well, such as <th> for table header.

Table Header

To designate a heading for a column or a row, us the <th> element. It create a cell to input data just like the table data element. Use the scope attribute to signify what the heading is for: col, row, colgroup, rowgroup. This tremendously helps screen readers.

Table Structure

We can use several elements and attributes to help organize the data. Some of those are: <caption>, <thead>, <tbody>, and <tfoot>

  • Table Caption: provide a caption or title for a table. 
    <table>
  <caption>Design and Front-End Development Books</caption>
  ...
</table>
  • Table Head, Body, & Foot: like the structure of a HTML page itself. 
    <table>

Combining Multiple Cells

Often cell combination is necessary but needs to occur without breaking the overall row and column layout. colspan and rowspan are used for this. They work on both the <td> and <th> elements.

<table>
  <caption>Design and Front-End Development Books</caption>
  <thead>
    <tr>
      <th scope="col" colspan="2">Item</th>
      <th scope="col">Qty</th>
      <th scope="col">Price</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Don’t Make Me Think by Steve Krug</td>
      <td>In Stock</td>
      <td>1</td>
      <td>$30.02</td>
    </tr>
    ...
  </tbody>
  <tfoot>
    <tr>
      <td colspan="3">Subtotal</td>
      <td>$135.36</td>
    </tr>
    <tr>
      <td colspan="3">Tax</td>
      <td>$13.54</td>
    </tr>
    <tr>
      <td colspan="3">Total</td>

    </tr>
  </tfoot>
</table>

Table Borders

Two properties are of great use when bordering tables: border-collapse and border-spacing.

  • Border Collapse Property: borders from neighbor cells will overlap each other. Value: collapse, separate, and inherit 
    table {
  border-collapse: collapse;
}
th,
td {
  border: 1px solid #cecfd5;
  padding: 10px 15px;
}
  • Border Spacing Property: This is kind of like a margin for each cell. The value of it will space out border of neighboring cells by its value. 
    table {
  border-collapse: separate;
  border-spacing: 4px;
}
table,
th,
td {
  border: 1px solid #cecfd5;
}
th,
td {
  padding: 10px 15px;
}

    NOTE: The border-collapse property must be set to separate (its default) for this to work.

Adding Borders to Rows

Border cannot be applied to <tr> elements or table structural elements. It requires a bit of thought.

Place a bottom border on all th and td elements. Then use :last-child to select the last tr element, target its td elements to then removes its borders.

table {
  border-collapse: collapse;
}
th,
td {
  border-bottom: 1px solid #cecfd5;
  padding: 10px 15px;
}
tfoot tr:last-child td {
  border-bottom: 0;
}

Table Striping

Alternate colors on rows (or columns if you're weird). Instead of manually placing classes on even or odd rows, use the :nth-child pseudo-class selector with an even or odd argument.

table {
  border-collapse: separate;
  border-spacing: 0;
}
th,
td {
  padding: 10px 15px;
}
thead {
  background: #395870;
  color: #fff;
}
tbody tr:nth-child(even) {
  background: #f0f0f2;
}
td {
  border-bottom: 1px solid #cecfd5;
  border-right: 1px solid #cecfd5;
}
td:first-child {
  border-left: 1px solid #cecfd5;
}

Aligning Text

Use the text-align property to align horizontally. Use the vertical-align property to align text vertically. This vertical property works only with inline and table-cell elements -- it won't work for block inline-block, or any other element levels. It accepts top, middle, and bottom as values.
