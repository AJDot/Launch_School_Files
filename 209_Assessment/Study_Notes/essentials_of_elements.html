document definition type tells the browser what version of HTML should be used.

This tells the browser what character set to use. It this is not done, special characters like the copyright symbol (©) may not render correctly.

Semantics

Most of what you will read in this section originated from learn.shayhowe.com.

Overview

Semantics give content meaning. In HTML, there are elements with, and elements without, inherent meaning. Without semantics tags have no value and would only be useful for styling. Semantic elements have a built-in value that tells other users (including yourself in the future) the purpose of that element. This makes the code easier understand and communicate. One great benefit of semantic markup is to functionalize screen readers, search engines, and other devices that depend on it to perform their tasks.

Non-Semantic Elements

Divisions and Spans

<div>s are the non-semantic block-level element and <span>s are the non-semantic inline-level element. Their sole purpose is to provide containers for styling a section of code. A <div> is used to style a large group of code as a block while a <span> is used to style a small section of text of within a block-level element. Most often a <div> and <span> will have a class or id attribute to target it with CSS. This name should have some kind of semantic meaning as well.

<b>, <strong>, <i>, <em>

strong is used semantically to give a strong importance to content. b should be used to stylistically offset text.

em is used semantically to placed a stressed emphasis on text - another type of increased importance. i is phrased as alternative voice or tone - as when speaking, a different voice may be used.

Article

The <article> element should be used to identify a stand-alone group of elements that could be extracted and used on it's own without loss of meaning.

Section

A <section> should be used for a group of related text. It may include a header, headings, and paragraphs similar to an <article>. I think of a <section >as very similar to an <article >but a <section >might not make sense if it were extracted from the page.

Aside

Use for sidebars, inserts, and other content tangential to the surrounding content. Instinctively an aside may be pushed off to one side but this is not a requirement.
