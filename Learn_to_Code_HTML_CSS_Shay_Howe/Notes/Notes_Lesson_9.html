Adding Media

Video, audio, and images. In the past, the process to add images and inline frames has always been relatively while incorporating video and audio... not so much. Now video and audio are easier to work with thanks to native support directly from HTML.

Adding Images

Use the <img> inline element. It is a self-containing element. Specify the image to be used with the src attribute to supply the url (relative or absolute).

To be used equally as often is the alt attribute. This describes the contents of an image and will help screen readers describe the image, search engines to find info and may be displayed if the image does not load for whatever reason.

<img src="dog.jpg" alt="A black, brown, and white dog wearing a kerchief">

Supported Image Formats

The most commonly supports formats online are gif, jpg, and png images.

  • jpg: quality images with high color counts and decent file size. Commonly used for photographs.
  • png: great for images with transparencies or low color counts. Commonly used for icons or background patterns.

Sizing Images

We can tell the browser how large an image will be before it is loaded so the browser may reserve space for it before it loads in. The helps increase the loading speed of a page. One simple way to do this is give the image a width and height directly in the img html.

The width and height can also be set using CSS. If width and height are declared in both HTML and CSS then CSS will take precedence.

Specifying either a width or height will cause the other dimension to adjust in order to maintain the native aspect ratio. If both the width and height are set then the default aspect ratio will not be maintained.

Positioning Images

By default, images are inline-level elements. We may change this by using CSS - float, display, and box model properties including padding, border, and margin.

Inline Positioning Images

Simply loading an image will cause it to be placed inline with whatever other elements are there. This may affect the height of the line the image is place on. This is usually undesirable. Images are typically display as block-level or floated.

Block Positioning Images

Add the display: block property: value to an image will make the image appear on its own line. Tada.

Positioning Images Flush Left or Right

Float an image to do this. This is in fact the original intention for the creation of the float property. Once done, everything will flow right up next to the image. Use margin, padding, border, and background to build a frame and give it some breathing room.

When to Use an Image Element vs. a Background Image

The <img> element within HTML is preferred when the image being used holds semantic value and its content is relevant to the content of the page.

The background or background-image property within CSS is preferred when the image is part of teh design/user interface and semantics aren't a factor.

Adding Audio

Now with HTML5 it is easy to add audio using the <audio> element. It accepts a src and requires a closing tag (unlike <img>).

Audio Attributes

The most popular ones are autoplay, controls, loop, and preload.

The autoplay, controls, and loop attributes are all Boolean attributes, meaning they don't require a stated value. Just type the word if you want it. The functions of most attributes are obvious.

The preload attribute can take three values:

  • none: nothing will preload
  • auto: info about the audio file will preload
  • metadata: any available metadata info will preload
  • attribute not present: all info about the audio file will load

Using a value of metadata or none is a good idea when the audio file is not essential to the page. It will conserve bandwidth and allow pages to load faster.

Audio Fallbacks & Multiple Sources

The most commonly supported file formats are ogg, mp3, and wav. For the best browser support we'll need to use some fallbacks. This is a little complex to do.

  1. Remove the src attribute form the audio element to define a new source.
  2. Use the <source> with a nested src attribute.
  3. Use the type attribute to quickly help the browser identify which audio types are available.
<audio controls>
  <source src="jazz.ogg" type="audio/ogg">
  <source src="jazz.mp3" type="audio/mpeg">
  <source src="jazz.wav" type="audio/wav">
  Please <a href="jazz.mp3" download>download</a> the audio file.
</audio>

Adding Video

Since HTML5, this is very similar to adding audio. Use the <video> element. All of the same attributes (src, autoplay, controls, loop, and preload) and fallbacks apply here, too.

In general, the best practice here is to include the controls Boolean attribute unless there is a good reason not to allow users to start, stop, or replay the video.

It's always good to specify and width and height using CSS as well. This will make sure you have the correct size to fit on the page and pages will load faster because space is allocated meaning the rest of the page doesn't have to wait for the entire video to load in order to render.

Customizing Audio & Video Controls

By default, each browser has its own controls used for audio and video. With a little JavaScript, these can be customized to fit your needs.

Poster Attribute

For video there is an additional poster attribute. We can specify an image, in teh form of a URL, to be shown before the video is played. Basically you can take a screen capture from the video itself that best represents the video. When used as a poster, it will be the image displayed before the play button is clicked.

Adding Inline Frames

iframe. This element accepts the URL (relative or absolute) of another HTML page within the src attribute value. It's like loading a page within your page. Often Google maps, YouTube, and others will be used inside an iframe to embed info on the page.

there are a few default values that can be adjusted using frameborder, width, and height in the HTML or border, width, and height in the CSS.

Semantically Identifying Figures & Captions

Do this using <figure> and <figcaption> elements. Use this when elements that are grouped together are self-contained and would not disrupt the page if the group was moved to a different location on the page. Ex: moving the image and its caption to the bottom of the page does not disrupt the content/meaning of the page.

<figure>
  <img src="dog.jpg">
  <figcaption>A beautiful black, brown, and white hound dog wearing kerchief.</figcaption>
</figure>
