Getting to Know CSS

The Cascade

All styles cascade from the top to the bottom of the style sheet. This will allow styles near the top to be overwritten by styles farther down.

Calculating Specificity

Every selector in CSS has a specificity weight.

Type selector | lowest specificity | 0-0-1
Class selector | medium specificity | 0-1-0
ID selector | high specificity | 1-0-0

Specificity Points

Values are not computed in base 10. ID selectors do not have a value of "100" and class selector don't count as "10". (We'll talk about this later.)

The specificity weight will override the cascading top-to-bottom nature. Ex: an ID selector will always overrule a type selector regardless of where it is place in the style sheet.

Combining Selectors

Specificity Within Combined Selectors

When selectors are combined, so are the specificity weights of the individual selectors. Count each different type of selector.

A selector of #id1.class1 type1.class2 type2 will have a specificity weight of 1-2-2

Layering Styles with Multiple Classes

One way to keep the specificity weights of selectors to a minimum is to be as modular as possible. Share similar style between similar elements as much as possible.

Example: For some buttons maybe they all are desired to have a font size of 16 pixels but the background color needs to vary depending on where the buttons are used.

<!-- HTML -->
<a class="btn btn-danger">...</a>
<a class="btn btn-success">...</a>
/* CSS */
.btn {
  font-size: 16px;
}
.btn-danger {
  background: red;
}
.btn-success {
  background: green;
}
