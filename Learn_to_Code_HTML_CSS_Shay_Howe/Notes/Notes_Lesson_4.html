Opening the Box Model

How Are Elements Displayed?

  • Block-level elements: occupy any available width, regardless of their content, and begin on a new line. Usually used for larger pieces of content like headings and structure elements.
  • Inline-level elements: occupy only the width their content requires and line up on the same line. Usually used for smaller bits of content like a word in a sentence.

Display

The exact was an element is displayed is determined by the display property and this may be overwritten. The most common values are block, inline, inline-block, and none.

block and inline do what you think they do. inline-block is weird. This allows an element to behave as a block-level element (accepts all box model properties) but will be displayed in line with other elements (and will not begin on a new line).

The Space Between Inline-Block Elements

Inline-block element are not always touching (unlike inline elements which do touch). This space can be annoying and can be removed. Read on...

Lastly, none will completely hide an element and render the page as if it never existed. Anything inside a display: none; element will also behave this way.

What Is the Box Model?

The box model states that everything is a rectangular box and may have width, height, padding, border, and margin.

Working with the Box Model

The total width can be calculated by adding up every component in the left-right direction.

margin-right + border-right + padding-right + width + padding-left + border-left + margin-left

And the total height is:

margin-top + border-top + padding-top + height + padding-bottom + border-bottom + margin-bottom

Width & Height

Width

Only relevant to block-level elements since inline and inline-block elements will change width depending on their content.

Height

Default height is determined by its content. For non-inline elements, you can override this by using the height property.

Sizing Inline-Level Elements

These will not accept the width and height properties. However, inline-block elements will accept them.

Margin & Padding

Default values may be applied by the browser. Use a CSS reset to get rid of these if necessary. Then we can work from the ground up.

Margin

The amount of space surrounding an element. They are present outside the the border of an element and are always completely transparent. Used to position objects or space elements out for readability. NOTE: vertical margins (top and bottom) are not accept by inline-level elements.

Padding

Similar to margin but lies inside the border. This property does work vertically on inline-level elements.

Margin & Padding on Inline-Level Elements

Margins only work horizontally (left and right). Padding works on all four sides. NOTE: vertial padding my bleed into the lines above and below the element.

Borders

Separate the margin and padding of an element. The border requires three values: width, style, and color.

Box Sizing

Until now the box model has been additive. We can now (CSS 3) use the box-sizing property to change how this works. It accepts 3 values content-box, padding-box, and border-box.

Content Box

This is the default value and makes it additive.

Padding Box

Alters the box model by including any padding property values with the width and height of an element. This means that if the width was 400px and the padding was 20px on all sides then the total width will still be 400px. In the Content Box this would appear as a width of 360px and padding of 20px on both sides. Margin and border would still be added to calculate the total width of the element.

Border Box

Now any border or padding values will be included within the width and height of the element. Ex: A border of 10px and padding of 20px on all sides of an element of width 400px will calculate the total width of the element as 400 px. In the Content Box version this would look like a border of 10px, padding of 20 px, and a width of 340px. Margin will still need to be added to get the total width of the element.

Picking a Box Size

border-box is generally the best to use. It makes our math easy to do. If we want an element to be 400px wide then set the width to 400px. Then whatever the padding and border are change to, the width of the element will remain 400px.

This has the added advantage of being able to mix values of different kinds. We can have a width of 40% and a padding of 20px and a border of 10px. So the border and padding will remain constant while the width will remain 40% of its parent.
