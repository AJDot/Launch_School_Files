Positioning Content

Positioning with Floats

This property removes an element from the normal flow and position it to the left or right of its parent element. Everything else on the page will flow around the floated element. Float accepts just a few values, usually left and right are used.

Floats May Change an Element's Display Value

The display value of block is required for a float element. So any element previously not a block element may be altered after it is floated.

For a 3-column row, we may do something like the following for the 3 sections that appear back-to-back in the html file.

section {
  float: left;
  margin: 0 1.5%;
  width: 30%;
}

Now each section will take up 33% of the width of the parent element.

Clearing & Containing Floats

The float property was intended for images and images only, not for positioning. There are some downfalls to using it in this way. Some styles will not render correctly if that element is next to or is a parent element of a floated element. Often margin and padding values aren't interpreted correctly, causing them to run into the floated element.

Sometimes content wraps around a float when you don't want it to do so...

Clearing Floats

To clear a float and have the next element appear on a new line we can use the clear property and set it to left, right or both. This needs to be applied to the element after the floated element.

Containing Floats

The outcome of containing a float vs clearing one are nearly the same. Though, contining floats does help to ensure that all of our styles will be rendered properly.

Floated elements must reside in a parent element that will act as a container that is in the normal flow. Below is the CSS for the parent element of the floated element.

.group:before,
.group:after {
  content: "";
  display: table;
}
.group:after {
  clear: both;
}
.group {
  clear: both;
  *zoom: 1;
}

This technique is often called a "clearfix" and is often written as such in most websites.

Positioning with Inline-Block

In addition to floats, we can position elements using the display property with the inline-block value

section {
  display: inline-block;
  margin: 0 1.5%;
  width: 30%;
}

Placing this code for our 3 sections we discussed earlier will almost work. The last section will get pushed to the next line. Because these are inline-block display elements they will have a single space between each other. This must be removed for this technique to work properly.

Removing Spaces Between Inline-Block Elements

There are several ways to do this.

First: put each new section element's opening tag on the same line as the previous section. When there is no space in the HTML then there will be no space when the elements are rendered.

<header>...</header>
<section>
  ...
</section><section>
  ...
</section><section>
  ...
</section>
<footer>...</footer>

Second: open an HTML comment directly after an inline-block element's closing tag. Then close it immediately before teh next inline-block element's opening tag. In essence this is "commenting out" any whitespace between the elements.

<header>...</header>
<section>
  ...
</section><!--
--><section>
  ...
</section><!--
--><section>
  ...
 </section>
 <footer>...</footer>

Creating Reusable Layouts

It is always best to write as modular as possible so that code may be reused elsewhere. Which layout works best, floats or inline-block elements, and why?

Author's approach: Use inline-block elements to create the grid - or layout - of a page and use floats when content needs to wrap around an element (as they were intended to do for images).

Currently flex- and grid-based properties are currently being developed in CSS.

Uniquely Positioning Elements

To place an element exactly where you want, the position element is needed.

The position property identifies how an element is positioned on a page and if it will appear in the normal flow. It is used with offset properties top, right, bottom, and left.

The default value for position is static, i.e. in the normal flow. relative or absolute are commonly used to override this.

Relative Positioning

This keeps the normal position of the element in the flow so that all other elements flow around as if it was there. Then the box offset properties can be used to move where that element is displayed.

Absolute Positioning

This will cause the element to not appear in the normal flow. The position the element would have been in will not be preserved. These elements will be positioned to their closest relatively positioned parent element.

Typically most positioning can be handled without the use of the position property and box offset properties.
