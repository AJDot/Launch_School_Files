Building Forms

We need forms to acquire input from users. this input could be anything from a name, to a job application, to... lots of other things. Here we will discuss how to create a form but not how to process a form as that is beyond the realm of this book.

Initializing a Form

To add a form to a page, use the <form> element. This identifies where on the page control elements will appear; it will also wrap all elements included with the form, like a div element.

The most common attributes for a form are the action and method. the action will contain the URL to which information included in the form will be sent for processing. The method attribute is the HTTP method browsers should use to submit the pform data.

Text Fields & Textareas

Text Fields

The primary element for this is the input element and denotes a single line of text. we will always set a type attribute to define what type of information is to be captured and the name attribute which is used as the name of the control for the server to use.

<input type="text" name="username" />

For the longest time text and password were the only two values for type. In HTML5 there are new values to choose. This add semantic meaning to a larger variety of inputs. Below are some input types:

  • color
  • date
  • datetime
  • email
  • month
  • number
  • range
  • search
  • tel
  • time
  • url
  • week
<input type="date" name="birthday">
<input type="time" name="game-time">
<input type="email" name="email-address">
<input type="url" name="website">
<input type="number" name="cost">
<input type="tel" name="phone-number">

Textarea

Accepts larger passages from user - multi-line. It has start and end tags (input is self-closing) but there is no type since textarea accepts only plain text.

Multiple Choice Inputs & Menus

The are:

  • Radio Buttons: need type, name, value, and checked item should be checked by default 
    <input type="radio" name="day" value="Friday" checked> Friday
<input type="radio" name="day" value="Saturday"> Saturday
<input type="radio" name="day" value="Sunday"> Sunday
  • Check Boxes: need type, name, value, and checked item should be checked by default 
    <input type="checkbox" name="day" value="Friday" checked> Friday
<input type="checkbox" name="day" value="Saturday"> Saturday
<input type="checkbox" name="day" value="Sunday"> Sunday
  • Drop-Down Lists: use the <select> and <option> elements. 
    <select name="day">
  <option value="Friday" selected>Friday</option>
  <option value="Saturday">Saturday</option>
  <option value="Sunday">Sunday</option>
</select>
  • Multiple Selections: Use the Boolean attribute multiple on a select element.

Form Buttons

Most commonly a form button is used to submit the form.

  • Submit Input: created using the <input> element with a type value of submit. The value will be what is displayed on the button. 
    <button name="submit">
  <strong>Send Us</strong> a Message
</button>
  • Submit Button: is not self-contained unlike an input and is used if more control over the structure and design is desired. Since it has opening and closing tags it can wrap other elements. 
    <button name="submit">
  <strong>Send Us</strong> a Message
</button>

Other Inputs

The input element has other uses as well

  • Hidden Input 
    <input type="hidden" name="tracking-code" value="abc-123">
  • File Input 
    <input type="file" name="file">

Organizing Form Elements

labels, fieldsets, and legends

Label

Provides captions or heading for form controls and ties them together for screen readers etc to use.

Labels may include a for attribute which should be the same as the id attribute on the form control the label corresponds to. In this way users can click on a label element to bring focus to the proper form control.

<label for="username">Username</label>
<input type="text" name="username" id="username">

The label element may wrap its intended form controls. In this case the for and id attributes are not needed.

<label>
  <input type="radio" name="day" value="Friday" checked> Friday
</label>
<label>
  <input type="radio" name="day" value="Saturday"> Saturday
</label>
<label>
  <input type="radio" name="day" value="Sunday"> Sunday
</label>

Fieldset

Group form controls and labels much like a <section> element. It is a block-level element used specifically inside a <form>.

<fieldset>
  <label>
    Username
    <input type="text" name="username">
  </label>
  <label>
    Password
    <input type="text" name="password">
  </label>
</fieldset>

Legend

Provides a caption, or heading, for the <fieldset> element. This element should come directly after the fieldset tag.

<fieldset>
  <legend>Login</legend>
  <label>
    Username
    <input type="text" name="username">
  </label>
  <label>
    Password
    <input type="text" name="password">
  </label>
</fieldset>

Form & Input Attributes

  • disabled: Boolean value used to turn of a control/element to make it unavailable for interaction. Disabled elements will not send any value to the server for form processing. If it is applied to a fieldset element then all form controls within it will be disabled. 
    <label>
  Username
  <input type="text" name="username" disabled>
</label>
  • placeholder: used on inputs or textareas. It is text that will disappear once the element is in focus. 
    <label>
  Email Address
  <input type="email" name="email-address" placeholder="name@domain.com">
</label>
  • required: a Boolean attribute that enfores an element or form control must contain a value upon being submitted to the server. An error message will pop up if a form is submitted without a value in all its required fields. This message is controlled entirely by the browser. However, invalid elements and form controls can be styled using the :optional and :required CSS psuedo-classes. 
    <label>
  Email Address
  <input type="email" name="email-address" required>
</label>
