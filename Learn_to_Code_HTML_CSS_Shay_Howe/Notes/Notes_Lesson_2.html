Getting to Know HTML

There is a right way and a wrong way to display content.

Semantics Overview

Semantics within HTML is the practice of giving content on the page meaning and structure by using the proper element. Semantic element help in several ways: enabling computers, screen readers, search engines, and other devices to adequately read and understand the content on a web page. It's also easier to work with; it clearly describes the content.

Divisions and Spans

Divisions are block-level and spans are inline-level.

<!-- Division -->
<div class="social">
  <p>I may be found on...</p>
  <p>Additionally, I have a profile on...</p>
</div>

<!-- Span -->
<p>Soon we'll be <span class="tooltip">writing HTML</span> with the best of them.</p>

Using Text-Based Elements

Headings

Use them. h1...h6 helps search engines to index and determine the content on a page. Heading levels should be used in that order. Each heading should be used where it is semantically valuable. Headings should not be used to make text look bold or big.

Bold Text with Strong

To place a strong importance on a word, bold text with <strong> instead of <b>. Use the <strong> element when you would actually say it differently. Use the <b> element when you're just styling something bold, like the title of a table column.

Italicize Text with Emphasis

The place emphasis on text use <em> instead of <i>. Use the <em> element when you would actually say it differently. Use the <i> element when you need to say something in an alternative voice or tone, like a quote inside a paragraph.

Creating Hyperlinks

Wrapping Block-Level Elements with Anchors

<a> is an inline element, and is typically used to wrap only inline-level elements. However, HTML 5 now specifically allows the anchor element to wrap either block-, inline-, or any other level elements. This allows entire blocks to become links.

Linking to an Email Address

To create an email link, the href attribute will need to start with mailto: followed by the email address. to add a subject line, include subject= parameter right after a ? following the email address. If there are multiple spaces in the address, use the space encoding %20.

To add body text, do the same this as subject= but use body= instead. Use & between parameters when using more than one.
